# User Test Enhance

UT 보고서의 개선 권고사항을 실제 코드/스펙에 반영한다.

## 입력 파싱

`$ARGUMENTS`를 파싱한다:
- 보고서 경로 (선택): 특정 보고서 지정. 없으면 가장 최근 보고서 사용
- `.user-test/report-*.md` 패턴으로 검색

## 실행 절차

### Step 1: 보고서 로드 및 분석

보고서를 읽고 개선 권고사항을 추출한다:
- P0 (즉시 수정) 항목 목록
- P1 (다음 스프린트) 항목 목록
- P2 (백로그) 항목 목록

### Step 2: 적용 범위 확인

사용자에게 개선 범위를 확인한다:

```
📋 보고서에서 추출한 개선 항목:

[P0] 즉시 수정 ({n}건)
  1. {항목} → 예상 수정 위치: {file/location}
  2. ...

[P1] 다음 스프린트 ({n}건)
  1. ...

[P2] 백로그 ({n}건)
  1. ...

어떤 범위까지 적용할까요?
  (a) P0만
  (b) P0 + P1
  (c) 전체
  (d) 직접 선택
```

### Step 3: 개선 실행

선택된 항목에 대해 순차적으로 수정을 적용한다:

1. 각 항목에 대해:
   - 관련 파일/코드를 찾는다 (Glob, Grep)
   - 수정 계획을 수립한다
   - 수정을 적용한다 (Edit)
   - 수정 사항을 사용자에게 보여준다

2. **spec 대상이었던 경우:**
   - 문서 수정 (문구 개선, 구조 변경, 누락 정보 추가)

3. **코드 대상이었던 경우:**
   - 코드 수정 후 빌드/린트 확인
   - 가능하면 테스트 실행

### Step 4: 변경 요약

```markdown
## 개선 적용 결과

| # | 항목 | 우선순위 | 수정 파일 | 상태 |
|---|------|----------|-----------|------|
| 1 | ... | P0 | `path/to/file` | ✅ 완료 |
| 2 | ... | P0 | `path/to/file` | ⚠️ 수동 확인 필요 |

### 미적용 항목 (수동 처리 필요)
- {항목}: {이유}
```

### Step 5: 재테스트 제안

> 개선이 적용되었습니다. 변경 효과를 검증하려면:
> `/user-test:start {target}`으로 동일 페르소나 그룹에 재테스트를 실행하세요.
