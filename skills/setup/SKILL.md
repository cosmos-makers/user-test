# User Test Setup

페르소나 그룹을 구성한다. 결과는 `.user-test/personas.json`에 저장된다.

## 입력 파싱

`$ARGUMENTS`를 파싱한다:
- 첫 번째 인자: 프리셋 이름 또는 `custom`
- 두 번째 인자 (선택): 페르소나 수 (기본값: 프리셋 기본 수)
- 인자 없으면: 대화형으로 선택

## 실행 절차

### Step 1: 프리셋 목록 로드

플러그인의 `presets/` 디렉토리에서 사용 가능한 프리셋을 읽는다:

```
${CLAUDE_PLUGIN_ROOT}/presets/*.json
```

각 프리셋의 `name`과 `description`을 사용자에게 보여준다.

### Step 2: 프리셋 선택 또는 커스텀 구성

**프리셋 선택 시:**
- 해당 프리셋의 페르소나를 로드
- 사용자가 수를 지정하면 랜덤 또는 선택적으로 추출
- 사용자가 속성 수정을 원하면 개별 조정 가능

**커스텀 구성 시 (`custom`):**
- 사용자에게 다음 속성을 질문하며 페르소나를 하나씩 구성:

```
인구통계: 연령, 성별, 거주지역, 소득수준, 직업
행동/경험: 도메인 숙련도, IT활용능력, 경쟁사 경험, 현재 대안
심리/상황: 기술수용도, 동기(Pain Point), 사용 맥락
```

- 최소 2명, 최대 10명
- 각 페르소나에 고유 `id`와 `label` 부여

**혼합 구성:**
- 프리셋을 기반으로 일부 수정하거나 커스텀 페르소나를 추가할 수 있다

### Step 3: 저장

구성된 페르소나 그룹을 `.user-test/personas.json`에 저장한다:

```json
{
  "preset": "프리셋명 또는 custom",
  "created_at": "ISO 날짜",
  "personas": [/* 페르소나 배열 */]
}
```

### Step 4: 확인

구성된 그룹을 테이블 형태로 보여주고 확인을 받는다:

| ID | Label | 연령 | IT숙련도 | 기술수용도 | 핵심 동기 |
|----|-------|------|----------|------------|-----------|

### Step 5: 페르소나 일기 생성

각 페르소나에 대해 "어제 하루" 일기를 생성한다. 이 일기는 페르소나에 생생한 경험적 맥락을 부여하여 테스트 시 더 현실적인 반응을 이끌어낸다.

**생성 규칙:**
- 각 페르소나당 **sonnet 모델 에이전트**를 **병렬로** spawn한다
- 에이전트에게 전달하는 프롬프트:

```
다음 인물의 "어제 하루"를 1인칭 일기체로 작성하라.

## 인물 프로필
{페르소나 전체 속성}

## 작성 규칙
1. 분량: 300-500자 (한국어 기준)
2. 이 사람의 직업, 생활환경, IT 활용 수준, 핵심 동기에 맞는 구체적 에피소드를 포함하라
3. 반드시 포함할 요소:
   - 하루 중 디지털 기기/서비스를 사용한 구체적 장면 1-2개
   - 그 과정에서 느낀 감정이나 불편함
   - 현재 대안({current_workaround})을 실제로 사용하는 장면
4. 이 인물의 말투와 사고방식을 반영하라
5. 인구통계적 고정관념이 아니라, 프로필에 정의된 행동 특성과 경험에 기반하라
6. 마크다운 형식으로 작성하라 (제목 없이 본문만)
```

**저장:**
- 각 일기를 `.user-test/diaries/{persona.id}.md`에 저장한다
- frontmatter:

```yaml
---
persona_id: {id}
persona_label: {label}
generated_at: {ISO 날짜}
---
```

**건너뛰기:**
- `.user-test/diaries/{persona.id}.md`가 이미 존재하면 해당 페르소나는 건너뛴다
- 사용자가 일기 생성을 원하지 않으면 이 단계 전체를 건너뛸 수 있다 ("일기 생성을 건너뛰시겠습니까?")

### Step 6: 완료 안내

> 페르소나 {N}명 구성 완료. 일기 {M}편 생성됨.
> 다음 단계: `/user-test:start [target]`으로 이 그룹에 UT를 실행하세요.
